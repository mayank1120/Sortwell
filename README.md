# Sortwell

Sortwell is a native macOS, local-first file organiser. It scans a user-selected folder locally, builds a review plan, applies only approved moves and Trash actions, and records a local undo journal.

## Download

Download the latest DMG from [GitHub Releases](https://github.com/mayank1120/Sortwell/releases), open it, and drag Sortwell into Applications.

Current prerelease builds are ad hoc signed but not Apple-notarised. On first launch, macOS may prevent the app from opening. In Finder, Control-click Sortwell, choose **Open**, then confirm **Open**. On some macOS versions you may instead need to choose **Open Anyway** under **System Settings > Privacy & Security**.

## Open and Run

1. Open `Sortwell.xcodeproj` in Xcode.
2. Select the `Sortwell` scheme and `My Mac` destination.
3. Run the app with `Command-R`.
4. Choose or drop a folder to start a local, read-only scan.
5. Open Settings with `Command-,` to configure local content analysis, OCR, recovery retention, categories, and classification rules.

## Regenerate the Project

The Xcode project is generated from `project.yml` using XcodeGen:

```sh
xcodegen generate
```

The app icon is generated reproducibly from the existing Sortwell mark:

```sh
swift Scripts/generate_app_icon.swift
```

## Verify

```sh
xcodebuild test \
  -project Sortwell.xcodeproj \
  -scheme Sortwell \
  -destination "platform=macOS" \
  CODE_SIGNING_ALLOWED=NO
```

Rendered review images are generated by `VisualSnapshotTests` in `VisualSnapshots/`. This developer-only test intentionally writes PNG design-review artefacts.

## Release Validation

`project.yml` defines version `1.0.0 (1)`, hardened runtime, the app icon, sandbox entitlements, and the development team. A local Release archive can be produced without distribution credentials:

```sh
xcodebuild archive \
  -project Sortwell.xcodeproj \
  -scheme Sortwell \
  -configuration Release \
  -destination "generic/platform=macOS" \
  -archivePath /tmp/Sortwell.xcarchive \
  CODE_SIGN_STYLE=Manual \
  CODE_SIGN_IDENTITY="-" \
  DEVELOPMENT_TEAM=""
```

Public distribution additionally requires a valid Developer ID Application certificate and `notarytool` credentials. Verify the exported app with `codesign --verify --deep --strict` and `spctl --assess`, then staple the successful notarisation ticket.

To build an ad hoc signed DMG for local testing or a GitHub prerelease:

```sh
Scripts/build_release_dmg.sh
```

The script creates `dist/Sortwell-<version>.dmg` and a SHA-256 checksum. These builds are not notarised and will trigger Gatekeeper warnings on other Macs.

## Safety Model

- Real folder access is explicit through the macOS folder picker and security-scoped access.
- Scanning is read-only and local.
- Supported document text, image metadata, and optional OCR are analysed locally without network access.
- Software-project folders and nested folder contents are protected from automatic reorganisation.
- Exact duplicate groups require SHA-256 plus byte comparison.
- Duplicate cleanup requires explicit per-group approval.
- Apply writes a local JSON undo journal before and after each action.
- Approved Trash actions receive a hidden recovery copy before macOS moves the original to Trash.
- Existing destination filenames are never overwritten; Sortwell picks an available destination name.
- Security-scoped bookmarks and journals allow eligible sessions to remain available after relaunch.
- Completed recovery data is pruned according to the retention period in Settings; interrupted sessions are retained for review.
